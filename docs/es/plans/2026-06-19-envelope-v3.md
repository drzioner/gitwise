## Resumen

Sprint 3 del roadmap post-0.28. Unifica el envelope `--json` de todos los
comandos al formato v3:

```json
{"v": 3, "ok": true, "command": "<cmd>", "data": {...}, "hints": [...], "errors": [...]}
```

Cambios incluidos (breaking change documentado, 0.31.0 -> 0.32.0):

- **Envelope v3**: los campos planos se anidan bajo `data`; se anaden
  `command`, `hints`, `errors` (siempre presentes).
- **status (S2+S3)**: nueva categoria `data.conflicted` (UU/AA/DD ya no se
  doble-cuentan en staged/unstaged) y `data.files` como objetos `FileEntry`
  (`{path, code, status, staged, binary, old_path?}`) en vez de strings planos.
- **log (J7+J9)**: `parents` y `stats` pasan a ser arrays; `data.total`/
  `truncated` + `hints` reportan paginacion.
- **diff (J3)**: per-file entries ya son objetos ricos (path/insertions/
  deletions/binary/status).
- **schema (M1+G5)**: `gitwise schema <cmd> --output` devuelve el schema del
  output (especifico para status/log/diff, fallback generico del envelope v3
  para el resto).

Las decisiones tomadas: big-bang breaking + docs de migracion (sin shim v2);
dos PRs secuenciales (este es Sprint 3; Sprint 4 se planifica aparte tras el
merge); S2+S3 incluidos.

El detalle completo (tareas T0-T14, contracts, antes/despues por comando) esta
en el archivo canonico en ingles.
