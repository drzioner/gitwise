# Sprint 3 — Envelope v3 — Implementation Plan

**Goal:** Unify `--json` across ~27 commands to `{"v":3,"ok","command","data","hints","errors"}` (nested `data`), refactor `status` (conflicted bucket + FileEntry), make `log.parents`/`stats` arrays, publish output schemas — as one documented breaking change (0.31.0 -> 0.32.0).

**Architecture:** `ok_envelope`/`error_envelope` are the single chokepoint. They require `command` positional and nest payload under `data`, so per-call-site migration is mechanical (prepend command name). A parametrized contract test validates invariants in one place and catches any unmigrated caller.

**Definition of Done (mechanical):**
1. `TZ=UTC uv run pytest` green (incl. `tests/test_envelope_contract.py`).
2. `ruff check`/`ruff format --check` clean; `uv run basedpyright` clean.
3. Every `--json` command emits exactly `{v:3, ok, command, data, hints, errors}`.
4. `gitwise schema status --output` returns a valid JSON Schema.
5. CHANGELOG has `### Breaking Changes` with before/after per command; ROADMAP baseline bumped.
6. `/multi-review` no blocking findings.

Decisions: big-bang breaking + migration docs; two sequential PRs (this is Sprint 3); S2+S3 included.

## Migration notes (v2 flat -> v3 nested)

Every `--json` output changed shape. Top-level command fields moved under
`data`; the envelope always carries `command`, `hints`, `errors`.

### Generic

```json
// BEFORE (v2)
{"v": 2, "ok": true, "commits": [...], "count": 3}
// AFTER (v3)
{"v": 3, "ok": true, "command": "log", "data": {"commits": [...], "count": 3, "total": 3, "truncated": false}, "hints": [], "errors": []}
```

### status (S2 + S3)

```json
// BEFORE
{"v": 2, "ok": true, "staged": 2, "unstaged": 1, "untracked": 0, "files": ["a.py", "b.py"]}
// AFTER
{"v": 3, "ok": true, "command": "status",
 "data": {
   "staged": 2, "unstaged": 1, "untracked": 0, "conflicted": 1,
   "files": [
     {"path": "a.py", "code": "M ", "status": "modified", "staged": true, "binary": false},
     {"path": "b.py", "code": "UU", "status": "conflict", "staged": false, "binary": false}
   ]
 }, "hints": [], "errors": []}
```

Conflicted files (UU/AA/DD) are no longer double-counted in `staged` +
`unstaged`; they appear only in `data.conflicted`.

### log (J7 + J9)

```json
// BEFORE
{"v": 2, "ok": true, "commits": [{"parents": "h1 h2", "stats": " 1 file changed, ..."}]}
// AFTER
{"v": 3, "ok": true, "command": "log",
 "data": {"commits": [{"parents": ["h1", "h2"], "stats": [{"path": "a.py", "insertions": 1, "deletions": 0, "binary": false}]}],
          "count": 1, "total": 1, "truncated": false}, "hints": [], "errors": []}
```

### Discovering the shape per command

Run `gitwise schema <command> --output` to get the v3 envelope JSON Schema
(specific for status/log/diff; generic envelope fallback for the rest).
